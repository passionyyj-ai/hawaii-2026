# TravelMate AI v21.0 변경사항

## 여행 생성·참가

- 여행 코드를 항상 `AB3Q-9LXT` 형식으로 생성하고 저장합니다.
- 하이픈 없이 붙여 넣어도 같은 형식으로 정규화한 뒤 조회합니다.
- Supabase에도 코드 형식 제약을 추가해 저장 형식이 다시 섞이지 않게 했습니다.
- QR 링크, 코드 복사, 직접 참가가 동일한 코드를 사용합니다.

## 일정·첨부파일 공유

- 중복되어 있던 일정 동기화 실행부를 제거하고 `v21.js`가 단독으로 담당합니다.
- 원격 일정 적용 후 같은 데이터로 계속 새로고침되는 현상을 세션 서명으로 차단했습니다.
- 로컬 수정 중 원격 조회가 겹치면 로컬 변경을 먼저 저장합니다.
- 첨부파일 조회·삭제를 현재 여행 코드 범위로 제한했습니다.
- Storage 공개 URL의 경로 문자를 안전하게 인코딩했습니다.
- 기존 기기 내부 일정·상세정보·첨부파일 이관 기능은 유지했습니다.

## iPhone Safari/PWA 음성 인식

- 한 번의 버튼 클릭에 하나의 `SpeechRecognition` 세션만 실행합니다.
- 자동 재시작과 별도의 `getUserMedia` 사전 점유를 제거했습니다.
- TTS를 먼저 중지한 뒤 사용자 클릭 이벤트 안에서 바로 음성 인식을 시작합니다.
- 영어·한국어 버튼 전환 시 기존 세션을 안전하게 종료합니다.
- `no-speech`, 권한, 마이크 점유, 네트워크 오류를 구분해 표시합니다.
- 시작이 8초 넘게 지연되면 자동 반복 없이 정상 상태로 돌아갑니다.

## 업데이트

- Service Worker의 `skipWaiting`, `clients.claim`, 자동 새로고침을 사용하지 않습니다.
- 새 버전은 앱이나 탭을 완전히 닫았다가 다시 열 때 안전하게 적용됩니다.

## 주의

현재 공유 방식은 로그인 대신 여행 코드를 접근 수단으로 사용합니다. 여행 코드와 공유 링크를 받은 사람은 공동 데이터를 볼 수 있으므로 공개 게시판에 올리지 마세요.
